# David Grimsley

Software Engineer at [Arvig](https://www.arvig.com) working at the intersection of enterprise software and physical network infrastructure. I build the APIs, middleware, and automation that connect business platforms to telecom hardware across multi-vendor ISP environments.

Outside of work, I build tools I find interesting — from astronomical calculation libraries to visual interfaces for LLMs.

## Projects

### [LLM-DAG-UI](https://github.com/dgrims3/LLM-DAG-UI)
A visual branching interface for Claude conversations. Renders chat as a directed acyclic graph so you can branch off from any message and explore parallel threads. Built with React 19, React Flow, and Express.  
**[Try it live →](https://llm-dag-ui.vercel.app)**

### [MoonLocatorLibrary](https://github.com/dgrims3/MoonLocatorLibrary)
A high-precision Java library for lunar position calculations — azimuth, altitude, rise/set times, illumination, distance, and more. Implements NREL and Meeus algorithms. Published on [Maven Central](https://central.sonatype.com/artifact/io.github.dgrims3/moon-library).

```gradle
implementation("io.github.dgrims3:moon-library:1.0.0")
```

### [Lunar Locator](https://play.google.com/store/apps/details?id=com.ll.LunarLocator)
Android app that tracks the moon's position for any location and time, displaying altitude, azimuth, distance, and a map-based trajectory. Powered by MoonLocatorLibrary.
**[Play Store Link →](https://play.google.com/store/apps/details?id=com.ll.LunarLocator)**

### [Interactive Public Trails](https://github.com/dgrims3/InteractivePublicTrails)
Android app that renders Minnesota state park trails on Google Maps from GeoPackage data. Features R-tree spatial indexing, activity filtering, and UTM-to-WGS84 reprojection. Built with Kotlin, Jetpack Compose, and Google Maps Compose.
